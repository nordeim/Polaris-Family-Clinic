I’ve implemented the requested next-step changes to the button semantics and the design tokens, and confirmed they are consistent with the static mockup direction while keeping build clean. Here is a concise, technical summary of what’s now in place and why it fixes the earlier issues.

1) UiButton: link-friendly, token-driven, visually consistent

File:
- [src/components/ui/button.tsx](src/components/ui/button.tsx:1)

Key behavior:

- Props:
  - Extends Mantine ButtonProps.
  - Adds:
    - href?: LinkProps['href']
    - variant?: 'primary' | 'outline' | 'ghost' | 'subtle' (token variants)
    - size?: 'sm' | 'md' | 'lg'
    - fullWidth?: boolean
    - children: ReactNode

- Class composition:
  - Always includes:
    - .ui-btn
  - Plus:
    - .ui-btn--primary / --outline / --ghost / --subtle
    - .ui-btn--sm / --md / --lg
    - .ui-btn--full when fullWidth is true
  - All defined in tokens.css and responsible for:
    - Pill shape
    - Padding
    - Shadow
    - Color

- Link semantics:
  - When href is provided:
    - Renders:
      - <Link href={href} legacyBehavior passHref>
          <Button component="a" className={classes} variant="subtle" radius="xl" ... />
        </Link>
    - This:
      - Uses a real anchor for navigation.
      - Keeps Mantine styling subdued so our .ui-btn* classes define the look.
  - When href is not provided:
    - Renders:
      - <Button className={classes} variant="subtle" radius="xl" ... />
    - Used for actual buttons (non-nav).

Impact:
- Fixes the invalid href-on-button pattern that appeared in the static HTML snapshot.
- Reduces “Mantine default” visual dominance; unified pill CTAs now match the static mockup direction and are controlled centrally.
