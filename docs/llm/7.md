I have implemented the core UI primitives and tokenized styling foundation for the landing and shared layout, aligned with the static mockup and architecture.

Implemented elements

1) Design tokens

- File: [src/styles/tokens.css](src/styles/tokens.css:1)

Purpose:
- Centralizes visual system:
  - Fonts (body + headings).
  - Colors (primary, backgrounds, borders, text, positive).
  - Radii, shadows, spacing.
- Defines utility classes used by UI primitives:
  - Buttons:
    - .ui-btn, size variants, primary/outline/ghost/subtle.
  - Cards:
    - .ui-card, .ui-card--elevated.
  - Badges:
    - .ui-badge, .ui-badge--soft, .ui-badge--outline.
  - Sections:
    - .ui-section, .ui-section--default/alt/cta.
  - Misc:
    - .nav-link, .field-select, .field-input, .step-number, .list-check, etc.

Alignment:
- Mirrors the feel of static/index.html:
  - Soft cards, pill buttons, high contrast, senior-friendly typography.

2) Global styles

- File: [src/styles/globals.css](src/styles/globals.css:1)

Key points:
- Imports tokens.css.
- Provides base reset:
  - Box-sizing, full-height layout.
- Sets:
  - Body font (var(--font-sans)).
  - Background color and text color from tokens.
- Ensures main has min-height 100vh.
- Keeps things minimal and consistent with the landing mockup.

3) UiButton

- File: [src/components/ui/button.tsx](src/components/ui/button.tsx:1)

Description:
- Thin wrapper over Mantine Button.

Behavior:
- Props:
  - variant: 'primary' | 'outline' | 'ghost' | 'subtle' (default: 'primary').
  - size: 'sm' | 'md' | 'lg' (default: 'md').
  - fullWidth?: boolean.
  - Inherits Mantine ButtonProps.
- Applies classes:
  - Base: ui-btn
  - Variant: ui-btn--primary / outline / ghost / subtle
  - Size: ui-btn--sm / md / lg
  - Full width: ui-btn--full
- Renders:
  - <Button className={classes} ...>{children}</Button>

Role:
- Stable, token-driven button primitive for:
  - Landing CTAs.
  - Core flows (/book, /profile, /staff/appointments).

4) UiCard

- File: [src/components/ui/card.tsx](src/components/ui/card.tsx:1)

Description:
- Wrapper over Mantine Card.

Behavior:
- Props:
  - elevated?: boolean.
  - Inherits CardProps.
- Applies classes:
  - ui-card
  - ui-card--elevated when elevated is true.
- Renders content via Mantine Card with consistent padding, border, and shadow.

Role:
- Used for:
  - Landing hero cards.
  - Why Us / How It Works / Staff previews, etc.

5) UiBadge

- File: [src/components/ui/badge.tsx](src/components/ui/badge.tsx:1)

Description:
- Wrapper over Mantine Badge.

Behavior:
- Props:
  - variant: 'soft' | 'outline' (default: 'soft').
  - Inherits BadgeProps.
- Applies:
  - ui-badge + ui-badge--soft | ui-badge--outline.

Role:
- For small labels:
  - “Designed for seniors”
  - Queue/appointment status hints, etc.

6) Section

- File: [src/components/ui/section.tsx](src/components/ui/section.tsx:1)

Description:
- Layout primitive for page sections.

Behavior:
- Props:
  - id?: string
  - variant: 'default' | 'alt' | 'cta' (default: 'default')
  - children: ReactNode
  - className?: string
- Renders:
  - <Box component="section" id={id} className="ui-section ui-section--{variant}" ...>

Role:
- Used to structure:
  - Hero.
  - Why Us.
  - How It Works.
  - Seniors.
  - Staff.
  - CTA.
  - Contact/footer.
